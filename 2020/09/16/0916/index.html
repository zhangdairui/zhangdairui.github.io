Nodejs连接mysql报错ERROR 1045 (28000)解决办法
emo | 2020年09月16日 | 搭建各种各样的环境

在MAC下，将node项目连接到本地mysql的时候，出现了下面的报错：

ERROR 1045 (28000): Access denied for user ‘root’@’localhost’ (using password: YES)

其实这个报错的最根本的意思其实就是密码不对。所以不要轻易去复制粘贴网上各种解决方案。
比如如下一个一传十十传百的解决方案：

ALTER USER ‘root’@’localhost’ IDENTIFIED BY ‘你的新密码’;
（有一些教程还会把你的新密码那里填成password或者1111等等）

这个时候如果不仔细看，直接按照步骤使用这个命令，很容易不小心又把密码更换错了，或者最后还是和你node项目里的app.js对不上

正确做法是要先检查app.js中，const connection = mysql.createConnection这个地方传入的user以及password到底与自己mysql设置的密码是否一致。可以在mysql的终端里多次尝试密码。
