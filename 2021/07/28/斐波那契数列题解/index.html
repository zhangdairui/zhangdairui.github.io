No.10

题目

代码

如果纯递归会超时（递归太深栈溢出）。
就直接准备一个数组记录下每一次计算完fibResult的结果。

/**
 * @param {number} n
 * @return {number}
 */
var fib = function(n) {
     var fibResult = new Array(n+1); //用来记录fib每一项的结果。
     fibResult[0]=0;
     fibResult[1]=1;

     for(var i = 2;i<n+1;i++){
        fibResult[i] = (fibResult[i-1]+fibResult[i-2])%1000000007;
     }

    return fibResult[n];

};

其实更好的方法是不用数组，准备3个参数，随着指针往后挪 变成三个元素之间的问题。
以下是leetcode上的解答：

var fib = function(n) {
    let n1 = 0, n2 = 1, sum;
    for(let i = 0; i < n; i++){
        sum = (n1 + n2) % 1000000007;
        n1 = n2;
        n2 = sum;
    }
    return n1;
};

No.11 青蛙跳台阶

代码

/**
 * @param {number} n
 * @return {number}
 */
var numWays = function(n) {
/*
2个台阶，2种走法。
3个台阶，3种走法。（1+2种方法，刚好是1个台阶的方法数+2个台阶的方法数）
4个台阶，5种走法。（2+3种方法，刚好是2个台阶和3个台阶的方法数之和）
5个台阶，8种走法。（3+5种走法，刚好是3个台阶和4个台阶的方法数之和）
……以此类推。
之所以会产生这样的斐波那契数列，原因是：
比如说现在有10级台阶，
那么青蛙要么从第8级蹦2阶到位，要么从第9级蹦1阶，就这两种选择。
所以第10级台阶的走法数量 = 第8级的走法数量 + 第9级的走法数量

*/
var record  = new Array(n);
record[0] = 1;
record[1] = 1;
record[2] = 2;
if(n>2){
    for(var i = 3; i<n+1; i++){
        record[i] = (record[i-1] + record[i-2]) % 1000000007 ;
    }
}

return record[n];
};
