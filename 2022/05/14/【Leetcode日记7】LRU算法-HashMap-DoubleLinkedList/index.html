使用HashMap + Double Linked List实现LRU算法的原理

LRU算法定义

LRU是Least Recently Used 近期最少使用算法。 内存管理的一种页面置换算法，对于在内存中但又不用的数据块（内存块）叫做LRU，操作系统会根据哪些数据属于LRU而将其移出内存而腾出空间来加载另外的数据。

Leetcode No. 146

题目

Design a data structure that follows the constraints of a Least Recently Used (LRU) cache.

Implement the LRUCache class:

  • LRUCache(int capacity) Initialize the LRU cache with positive size capacity.
  • int get(int key) Return the value of the key if the key exists, otherwise return -1.
  • void put(int key, int value) Update the value of the key if the key exists. Otherwise, add the key-value pair to the cache. If the number of keys exceeds the capacity from this operation, evict the least recently used key.

The functions get and put must each run in O(1) average time complexity.

Example 1:

**Input**

["LRUCache", "put", "put", "get", "put", "get", "put", "get", "get", "get"]
[[2], [1, 1], [2, 2], [1], [3, 3], [2], [4, 4], [1], [3], [4]]


**Output**
[null, null, null, 1, null, -1, null, -1, 3, 4]

**Explanation**

LRUCache lRUCache = new LRUCache(2);
lRUCache.put(1, 1); // cache is {1=1}
lRUCache.put(2, 2); // cache is {1=1, 2=2}
lRUCache.get(1);    // return 1
lRUCache.put(3, 3); // LRU key was 2, evicts key 2, cache is {1=1, 3=3}
lRUCache.get(2);    // returns -1 (not found)
lRUCache.put(4, 4); // LRU key was 1, evicts key 1, cache is {4=4, 3=3}
lRUCache.get(1);    // return -1 (not found)
lRUCache.get(3);    // return 3
lRUCache.get(4);    // return 4

代码

//LRU: 如果数据被访问过，那么将来被访问的几率就更好。反之，一个数据很久都没有被访问过，就说明它不是很重要可以被洗掉

//重要：
//1. HashMap只要有key就可以访问到value，所以使用它查询快。但是它在内存中是无序的，不方便添加和删除
//2.双向链表使得本来无序的hashmap链接起来，让我们可以访问最初或者最后一个node，使整个数据结构中的内容变得有序。
//3. 为什么链表节点也要有key？因为当我们从链表中删除node的时候，也要通知HashMap，通过key去梦幻联动。
//4. 为什么要用双向链表？因为双向链表方便删除（移动）一个节点。只需要把prev指向next。如果是单向的链表，还需要从头遍历。这样就会又慢了。

public class LRUCache {

  //STEP ONE----------------------先制作双向链表

  //1. 所用的节点。每个节点包括：<key, value>,next和prev
  class DLinkedNode {
    int key,value;
    DLinkedNode prev;
    DLinkedNode next;
  }

  //2. 向链表中添加节点： 添加到head的下一个。head的下一个节点永远是最新被访问过的节点。
  private void addNode(DLinkedNode node) {
    node.prev = head;
    node.next = head.next;

    head.next.prev = node;
    head.next = node;
  }


  //3. 删除（可以理解为拿走）链表中的某个节点
  private void removeNode(DLinkedNode node){

    DLinkedNode prev = node.prev;
    DLinkedNode next = node.next;

    prev.next = next;
    next.prev = prev;
  }


  //4. 将一个指定的节点移动到head的下一个（这个节点是最近被使用的节点）
  private void moveToHead(DLinkedNode node){
    removeNode(node);
    addNode(node);
  }


  //5. 将链表的尾部节点删除。（最不常访问的落灰了的节点）
  private DLinkedNode popTail() {
    DLinkedNode res = tail.prev;
    removeNode(res);
    return res;
  }


  //STEP TWO----------------------制作HashMap+LinkedList
  private Map<Integer, DLinkedNode> cache = new HashMap<>();
  private int size; //目前储存的数据长度
  private int capacity; //整个内存的限制容量
  private DLinkedNode head, tail;//准备head和tail



  //STEP THREE-----------------制作LRU算法
  public LRUCache(int capacity) {
    //初始化LRU结构
    this.size = 0;
    this.capacity = capacity;

    head = new DLinkedNode();
    // head.prev = null;

    tail = new DLinkedNode();
    // tail.next = null;

    head.next = tail;
    tail.prev = head;
  }

//========================查找方法========================
  public int get(int key) {
    //当访问某个点时，（通过key查找）访问到了之后返回对应value并且别忘了将这个点更新到链表的头部。
    DLinkedNode node = cache.get(key);
    if (node == null) return -1;

    moveToHead(node);

    return node.value;
  }

//========================添加（删除）方法=======================

  public void put(int key, int value) {
    //当放置新的点（put）的时候，要看看已有的key对应的数据在内存中是个什么状态
    DLinkedNode node = cache.get(key);

    //如果说还没有这个点的话，那么我们可以把它new出来，就用传入的key和value
    if(node == null) {

      DLinkedNode newNode = new DLinkedNode();
      newNode.key = key;
      newNode.value = value;

      cache.put(key, newNode);
      addNode(newNode);

      ++size;

      if(size > capacity) {

        // 如果说在添加了之后size已经超出了内存，需要把尾部的节点删除掉
        DLinkedNode tail = popTail();
        cache.remove(tail.key);


        --size;

      }

    } else { 
      // 如果说这个点已经存在了，那更新一下value（代表使用过了）
      node.value = value;
      moveToHead(node);//再把它挪到最新的地方
    }
  }
}
