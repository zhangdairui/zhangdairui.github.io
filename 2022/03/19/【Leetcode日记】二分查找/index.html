关于Leetcode No.69 Sqrt(x)

这道题就是实现一个求平方根的Sqrt(x)函数。官方给出了4种题解，其中Approach 2就是Binary Search。这个二分查找每次看到都觉得“这题我肯定会！不就是对半砍掉么”但是一涉及到细节就很烦。所以在这篇记录一下二分查找的模板。

原题

Given a non-negative integer x, compute and return the square root of x.
Since the return type is an integer, the decimal digits are truncated, and only the integer part of the result is returned.
Note: You are not allowed to use any built-in exponent function or operator, such as pow(x, 0.5) or x ** 0.5.
Example 1:

Input: x = 4
Output: 2

Example 2:

Input: x = 8
Output: 2

Explanation: The square root of 8 is 2.82842…, and since the decimal part is truncated, 2 is returned.

  • Constraints: 0 <= x <= 231 - 1

Binary Search官方题解（Java）

class Solution {
  public int mySqrt(int x) {
    if (x < 2) return x;

    long num;
    int pivot, left = 2, right = x / 2; 
    //上面right=x/2是因为第一次直接就砍半之后取左边就行。因为这是找平方根，不可能在右半了。

    while (left <= right) {
      pivot = left + (right - left) / 2; //进入循环找到中点
      num = (long)pivot * pivot; // 看看中点的平方是多少
      if (num > x) right = pivot - 1; //如果大于target，说明我们需要把右侧边界缩小到中点左边一个
      else if (num < x) left = pivot + 1; //反之移动左边界
      else return pivot; //如果找到了直接返回pivot
    }
    //如果循环结束了，区间没有了，
    return right;
    //【为什么区间没有了之后 return的是right】：
    //【首先，最终导致left>right循环结束的主要原因，是发现中点的平方超出了目标，进而向左移动了右边界。结果导致区间不存在了。所以只要返回刚才的右边界就是最终的结果。】
    //【也可以理解为因为这道题要求会舍去小数点后面，取整数部分】
    //【比如找65的平方根，最终结束的瞬间，搜索区间肯定是[9,8]（left>right），这时候取8。取小的那一侧。】
    //【也就是说，这道题要求我们找一个数的平方根，宁可“不够”，也不能“超出”。所以最终从left和right里选择较小的一方】
  }
}

关于二分查找

框架

二分查找需要设定一个查找范围，也就是左侧边界和右侧边界。假如在一个数组里查找一个数值，那起始的左侧边界就是0（第一个索引所在位置），右侧边界显然就是length-1的最后一个索引所在位置。之后还需要一个中点， 然后在开始查找的过程（就是while循环）中，不断地根据中点的情况，将中点的值赋给左或者右边界，实现折半的效果。

如果不采取这样另设索引+while循环的方式的话，递归调用也是可以的。每次调用都传入新的折半之后的数组（可能空间占用太大了），或者新的左右边界。递归就不需要使用while循环，但是要注意出口的写法。

整体思路

int binarySearch(int[] nums, int target) { //输入正序数组和查找目标
    int left = 0, right = ...; //左右边界

    while(...) { //进入循环（开始查找）
        int mid = (right + left) / 2; //每次都找到新的中点，拿这个中点位置的数字和target做比较
        if (nums[mid] == target) { //当找到了（相等）
            ...
        } else if (nums[mid] < target) {//当该中点值小于目标点
            left = ... //移动左边界到中点-1的位置。
        } else if (nums[mid] > target) {//当该中点值大于目标点
            right = ...//移动右边界
        }
    }
    //出循环了（没找到）
    return ...;
}

具体案例-非递归

import java.util.Scanner;

public class BinarySearch {
    public static void main(String[] args) {
        int arr[] = {1,5,7,10,15,20,28,56,79,85}; //正序数组
        Scanner sc = new Scanner(System.in); //输入一个查找的target
        int val = sc.nextInt();
        int index = search(arr,val); //调用search函数
        System.out.println(index);
    }

    /**
     *
     * @param arr
     * @param val 要查找的数
     * @return 元素下标
     */
    private static int search(int[] arr, int val) {
        int right = arr.length - 1;
        int left = 0;
        while(left <= right) {
    //当left<=right，可以进行查找。
    //必须要带等号，因为即便区间是[2,2]，也是要看看2是不是的。
            int middle = (right + left) / 2;
            if (val == arr[middle]) {
                return middle;//找到则返回下标
            } else if (val < arr[middle]) {
                right = middle - 1;//要找得数比中间数小则在左边找。
            } else {
                left = middle + 1;//要找得数比中间数小则在右边找
            }
        }
        return -1;
    }
}

具体案例-递归

public class BinarySearchDemo {
    public static void main(String[] args) {
        int arr[] = {1,5,7,10,15,20,28,56,79,85};
        int index = binarySearch(arr,0,arr.length - 1,10);
        System.out.println(index);
    }

    /**
     *
     * @param arr
     * @param l 左下标
     * @param r 右下标
     * @param data  数据
     * @return
     */
    private static int binarySearch(int[] arr, int l, int r, int data) { //递归同时也传入边界
        int mid = (l + r) / 2;
        if(r < l){ //递归出口，当右边界小于左边界时，区间不存在了，可以停止查找。
            return -1;
        }
        if(arr[mid] == data){
            return mid;
        }
        if(arr[mid] < data){
            return binarySearch(arr,mid+1,r,data);
        }else{
            return binarySearch(arr,l,mid-1,data);
        }
    }
}

总结

Leetcode里的69这道题的二分查找解法，我认为难点在于循环结束之后返回值。比起基础框架的“找不到返回-1”，这里是“找不到整数的话就返回小的那一侧”。这样很容易就误以为返回的是left，忽略了最后是因为left>right才终止的查找。
