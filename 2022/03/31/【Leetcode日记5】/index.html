No.19. Remove Nth Node From End of List

题目

题解

（官方solution）

/**
 * Definition for singly-linked list.
 * public class ListNode {
 *     int val;
 *     ListNode next;
 *     ListNode() {}
 *     ListNode(int val) { this.val = val; }
 *     ListNode(int val, ListNode next) { this.val = val; this.next = next; }
 * }
 */



class Solution {
    public ListNode removeNthFromEnd(ListNode head, int n) {

    //新建一个dummy节点，使之与head相连。
    ListNode dummy = new ListNode(0);
    dummy.next = head;

    //计算出head链表的长度，为了操作链表，新建一个first指向head的头部

    int length  = 0;
    ListNode first = head;
    while (first != null) {
        length++;
        first = first.next; //循环向后移动first，直到末尾。此时已经获得length。
    //此时first也处于链表末尾。
    }


    length -= n;
    first = dummy; //使first再指向dummy

    while (length > 0) {
        length--;
        first = first.next;
    }//此时first指向了需要删除的节点的前驱节点 下一步只要改变next就可以

    first.next = first.next.next; //改变锁链

    return dummy.next; //返回从原来head位置开始的链表
}


    }

总结

代码中的要点：

  • dummy的存在意义：如果没有dummy的话，当我们需要删除的点就是head的时候就会出现问题。除非额外做一些判断。归根结底是因为我们想要删除链表中的某一个点，需要该点的前面的点，而head是没有前面的点的，所以我们要给他创造一个前面的点。从而把head节点变成一个有前有后的普通节点。最后也要return dummy.next而不是head。

  • first的存在意义：我们为了获得length，是不能直接操作head头部指针的，因为我们得留着“最初的锁链”，让他从中间断开，不能从头全都改变。另外，first更像是一个游标，改变first本身是无所谓的，随时可以再改回来，但是一旦改变first.next，就相当于改变了节点之间的连接，head也好dummy也好会一起更改。

对链表的加深理解

  • 链表是一种链式表达，关键就在于【链】。一个链表有val和next。val代表当前指针指向的节点的值，next代表下一个节点（一个指针）。无论是val还是next其实都是指针。当修改了链（当前元素的下一个元素的指向），才是真正意义上的修改了链表。
  • 在上面的代码中： ListNode dummy = new ListNode(0);这是最常用的初始化链表节点的方式，指针指向了dummy。然后使用dummy.next = head;就可以将dummy与head连上。而ListNode first = head;就是不一样的，first这里只是一个指针！只是创建了一个first链表指针！没有使用first.next将其与任何节点相连。这句代码的意思只是把first指针指向了head的位置。 相当于在head上面新建了一个游标！
