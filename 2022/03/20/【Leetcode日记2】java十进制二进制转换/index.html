No.191. Number of 1 Bits

原题

Write a function that takes an unsigned integer and returns the number of ‘1’ bits it has (also known as the Hamming weight).
Note:
Note that in some languages, such as Java, there is no unsigned integer type. In this case, the input will be given as a signed integer type. It should not affect your implementation, as the integer’s internal binary representation is the same, whether it is signed or unsigned.
In Java, the compiler represents the signed integers using 2’s complement notation. Therefore, in Example 3, the input represents the signed integer. -3.

Example 1:

Input: n = 00000000000000000000000000001011
Output: 3
Explanation: The input binary string 00000000000000000000000000001011 has a total of three '1' bits.

Example 2:

Input: n = 00000000000000000000000010000000
Output: 1
Explanation: The input binary string 00000000000000000000000010000000 has a total of one '1' bit.

Example 3:

Input: n = 11111111111111111111111111111101
Output: 31
Explanation: The input binary string 11111111111111111111111111111101 has a total of thirty one '1' bits.

*
Constraints:
The input must be a binary string of length 32.*

题解

public class Solution {
    // you need to treat n as an unsigned value
    public int hammingWeight(int n) {
        //=====下方已通过，采用了【转化为字符串】的方式=====
//         int result = 0;
//         String nString = Integer.toBinaryString(n);  //将十进制数转成字符串，例如n=5 ，s = "101"
//         char[] array = nString.toCharArray();

//         for(int i = 0 ; i<array.length; i++){
//             if (array[i] == '1') result++;
//         }

//         return result;
        //=======================================

        //=====下方已通过，采用了【二进制移位】的方式=====
        int result = 0;
        for(int i = 32 ; i>0 ; i--){
             if((n >>> i & 1) == 1) result++;
        }
        return result;
        //=======================================
    }
}

移位操作

所有的移位操作都是在32位的二进制数的基础之上操作的。

一个正数4，它在计算机内是00000000000000000000000000000100。

如果是负数和零，是用原码的补码表示的。

因此，利用移位操作将10进制转换为二进制其实就等同于：
虽然传入了一个十进制数，但我们需要将计算机内储存着的对应的二进制数表示出来放在明面上。

符号 含义
<< 左移运算符，将运算符左边的对象向左移动运算符右边指定的位数（在低位补0）
>> “有符号”右移运算 符，将运算符左边的对象向右移动运算符右边指定的位数。使用符号扩展机制，也就是说，如果值为正，则在高位补0，如果值为负，则在高位补1.
>>> “无符号”右移运算 符，将运算符左边的对象向右移动运算符右边指定的位数。采用0扩展机制，也就是说，无论值的正负，都在高位补0

左移运算符的代码示例：

左移运算符用“<<”表示，是将运算符左边的对象，向左移动运算符右边指定的位数，并且在低位补零。

其实，向左移n 位，就相当于乘上2 的n 次方，

例如下面的例子。

public class test{
    public static void main(String[] args) {
        int a=2,b=2,c;
        c=a<<b; //2的二进制是00000010 向左移动2位，就是000010000 变成了8（也就是乘上2的2次方）
        System.out.print("a 移位的结果是"+c);
    }
}
//输出结果：8

###利用无符号右移获取二进制

那么如何告诉计算机我们需要打印十进制n的二进制数呢？

只要应用位移操作运算符，计算机就会自动以二进制进行操作，

我们只需要想办法读取二进制n的每一位。

所以就从最高位开始，无脑右移。最后和1做一个&运算，

是1是0都趁机打印出来就可以。

循环32次，逐一读取每一位是1还是0.


public void binaryToDecimal(int n){
      for(int i = 31;i >= 0; i--)
          System.out.print(n >>> i & 1); //只有1&1才会输出1
 }
//如果是负数，会按照它的补码输出的。负数的补码 = 原码取反+1，符号位不变。
